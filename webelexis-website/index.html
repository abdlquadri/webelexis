Webelexis taking Elexis to the Web

Möchten Sie Ihren Patienten ermöglichen, selbst übers Internet Termine zu vereinbaren? Vielleicht ist Webelexis eine Lösung für Sie.

Für einen kurzen Einblick klicken Sie einfach hier in die Live-Simulation. Besuchen Sie die Seite (http://webelexis.ch:2015) bitte auch mit Smartphone oder Tablet, wenn vorhanden, damit Sie sehen, wie es auf solchen Geräten funktioniert. Wenn Sie neugierig geworden sind, lesen Sie bitte hier weiter.

1 Sie möchten Webelexis nur kurz ausprobieren?

Der Vorteil dieser Methode ist: Sie brauchen ausser Java nichts auf Ihrem Computer zu installieren, und webelexis-Agenda wird nach Programmende keinerlei Spuren im System hinterlassen. Der Nachteil ist: Jeder einzelne Start dauert relativ lang, da jedesmal benötigte Module aus dem Internet heruntergeladen werden müssen.

Sie benötigen:

  • Einen Computer mit Windows-, Linux- oder MacOS - Betriebssystem"
  • Java Runtime (JRE) 1.7 oder höher. Diese können Sie bei Oracle herunterladen. Allerdings haben Sie, wenn Sie Elexis-Anwender sind, Java höchstwahrscheinlich ohnehin schon installiert.
  • Das Webelexis fat-jar.

Laden Sie config.json herunter und öffnen Sie die Datei in einem Text-Editor (Nicht in Microsoft Word(tm), sondern in einem reinen Texteditor, z.B. Notepad). Passen Sie folgende Zeilen an:

  • "connection": "MySQL" - Wenn Sie einen PostgreSQL Server haben, schreiben Sie hier "PostgreSQL"
  • "host": Schreiben Sie hier die IP Adresse oder den Computernahmen des Computers, auf dem die Elexis-Datenbank läuft. Das ist üblicherweise der Server.
  • "username": Tragen Sie hier den Datenbank-Usernamen ein. Also den usernamen, mit sich Ihre Elexix-Instanzen mit der Datenbank verbinden. (Nicht etwa Ihr Elexis-Login-Name)
  • "password": Auch hier: Das Passwort der Datenbankanmeldung ist gefragt, nicht das Passwort, der Elexis-Anmeldung.
  • "database": Der Name Ihrer Elexis Datenbank. Meistens lautet dieser "elexis".
  • Unter "Agenda" passen Sie die Zeile "resource" an den Namen des gewünschnten Agenda-Bereichs an. Den Rest können Sie vorerst so lassen

Starten Sie dann das Programm mit folgendem Befehl:

java -jar base-module-x.y.z-fat.jar -conf config.json

(Wobei Sie für x.y.z die korrekten Versionsnummern einsetzen müssen)

Sobald das Programm gestartet ist, sollten Sie mit ihrem Lieblingsbrowser nach localhost:2015 gehen können, und sehen die öffentliche Agenda.

2 Sie möchten Webelexis intensiver testen?

Der Vorteil dieser Methode ist: Abhängigkeiten werden nur einmal heruntergeladen und verbleiben auf dem Computer. Jeder weitere Start geht daher viel schneller, als der erste. Der Nachteil ist: Sie müssen vert.x installieren, und es werden einige Dateien auf Ihrem Computer installiert, die Sie ggf. manuell wieder löschen müssen.

Sie benötigen:

  • Dasselbe, wie unter 1 genannt.
  • das vert.x framework, das Sie hier herunterladen können.
  • Das gezippte Webelexis-Modul

Laden Sie config.json herunter und modifizieren Sie es wie bei (1) erklärt. Starten Sie dann das programm mit:

vertx runzip ch.webelexis~base-module~x.y.z.zip -conf config.json

Für x.y.z müssen Sie die richtige Versionsnummer einsetzen. Und beachten Sie auch die beinden Tilden (~) im Dateinamen; es sind nicht alles Bindestriche.

Auch hier sollten Sie nach erfolgtem Start das Agenda Programm mit localhost:2015 anzeigen können.

3 Sie möchten Webelexis selber bauen und ggf. modifizieren?

Der Vorteil dieser Methode ist: Sie können Webelexis an Ihre Bedürfnisse anpassen. Das ist - Bei Anpassungen in begrenztem Rahmen, wie etwa Änderung der angezeigten Texte etc. - nicht sehr schwierig. Mit etwas Programmierkenntnissen können Sie das Programm sogar vollkommen verändern.

Sie benötigen:

  • Einen Computer mit Linux oder MacOS, oder einen Windows-Computer mit VirtualBox und einer Linux-VM. Folgen Sie dieser Anleitung, um eine solche VM zu erstellen. (Selbstverständlich können Sie dieses Konzept auch für die Entwicklung unter Mac oder Linux verwenden, wenn Sie die Entwicklungsumgebung strikt vom System trennen möchten)
  • Ein Java Development Kit (jdk) 1.7 oder höher, das Sie z.B. bei Oracle beziehen können

Weitere Informationen, um eigene Verbesserungen und Anpassungen an Webelexis durch zuführen, finden Sie auf der Github-Seite. und im Wiki.

Dieses Modul ist erst der Anfang. Geplant sind eine ganze Reihe von Zugriffsmöglichkeiten auf Elexis für autorisierte Anwender über verschlüsselte Kommunikationswege. Wenn Sie Webelexis ausprobiert haben, bin ich sehr an Rückmeldungen interessiert. Insbesondere auch von Leuten, die Elexis auf PostgreSQL Datenbanken und unter Windows laufen haben, da ich nur unter Mac und Linux mit MySQL testen konnte. Weitere Informationen finden Sie auf Github Wenn Sie Webelexis in einer produktiven Umgebung einsetzen möchten, sollten Sie sich um die Absicherung des Systems Gedanken machen. Es ist empfehlenswert, hierzu den Rat einer Fachperson einzuholen.
